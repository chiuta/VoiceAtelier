Cele trei niveluri explicate

Vocile Starter (Web Speech API built-in) nu necesită nicio configurație și funcționează offline, dar sună robotic față de standardele moderne. Vocile Storyteller sunt modelele neurale care alimentează Microsoft Read Aloud și accesibilitatea Apple — în Edge pe Windows se deblochează 300+ voci publicabile gratuit. Uneltele Studio Pro necesită un cont gratuit sau instalare locală, dar produc audio indistinguibil de narațiunea profesională.

Criterii de selecție a vocii

Evaluează vocile pe: naturalețe (respirație, ritm, prosodie), claritate (consoane clare la viteză normală), consistență (caracter stabil pe 10+ minute) și gamă emoțională. Audiționează întotdeauna cu textul tău real.

Flux de lucru pentru narațiune de lungă durată

Folosește CLI-ul edge-tts pentru a procesa în lot toate capitolele în fișiere MP3, apoi editează în Audacity. Procesează la -23 LUFS cu filtrul loudnorm din ffmpeg pentru conformitate ACX/Audible.

Pentru conținut românesc: Carmen (neural) a Amazon Polly și vocile Alina/Emil ale Microsoft Edge sunt cele mai bune opțiuni gratuite. Disponibil prin edge-tts --voice ro-RO-AlinaNeural.

Export și distribuție

ACX (Audible) necesită WAV/MP3 la minimum 192 kbps, -23 LUFS, -3 dBFS peak. Toate uneltele Studio Pro produc fișiere procesabile cu ffmpeg, gratuit.

The Three Tiers Explained

Starter voices (Web Speech API built-ins) require zero setup and work offline, but sound robotic by modern standards. Storyteller voices are the neural models powering Microsoft Read Aloud and Apple accessibility — in Edge on Windows you unlock 300+ publishable voices for free. Studio Pro tools require a free account or local installation but produce audio indistinguishable from professional narration.

Voice Selection Criteria

Evaluate voices on: naturalness (breathing, rhythm, prosody), clarity (crisp consonants at normal speed), consistency (stable character across 10+ minutes), and emotional range (distinguishing dialogue from narration). Always audition using your actual manuscript text.

Workflow for Long-Form Narration

Use the edge-tts CLI to batch-process all chapters into MP3 files, then edit in Audacity for pacing and silence trimming. Process to -23 LUFS with ffmpeg's loudnorm filter for ACX/Audible compliance.

For Romanian content: Amazon Polly's Carmen (neural) and Microsoft Edge's Alina/Emil voices are the strongest free options. Available via edge-tts --voice ro-RO-AlinaNeural.

Export & Distribution

ACX (Audible) requires stereo or mono WAV/MP3 at minimum 192 kbps, -23 LUFS loudness, -3 dBFS peak. All Studio Pro tools produce files processable with ffmpeg at no cost.

1. Data Controller

2. Data Collected

VoiceAtelier does not store personal data. There are no registration forms, analytics systems, tracking cookies, or persistent server logs with IP addresses.

In normal operation two types of external requests occur: (a) fonts loaded from Google Fonts on first access; (b) when any neural voice is played from the "All Languages" tab, the selected sample text and voice name are sent over HTTPS to the relay alexio.tf/tts, which generates MP3 audio via edge-tts and returns the stream. The relay does not persist submitted text beyond the duration of the request.

3. Voice Synthesis

Two synthesis modes exist:

  • Local (Starter, Storyteller): text is processed via the Web Speech API, native in the browser. The text does not leave your device (exception: if your browser sends internally to a server, that is the browser vendor's responsibility).
  • Via relay (All Languages): text + voice name are sent to alexio.tf/tts. The response is an audio stream. We do not retain the text.

You can avoid the relay mode by using only the Starter and Storyteller tabs, or by running edge-tts locally via the CLI command shown in the Guide.

Application Architecture

VoiceAtelier is a single-file HTML with no backend, no database, no external JavaScript frameworks. It contains only HTML5, CSS3, and vanilla JavaScript.

External Dependencies

  • Google Fonts — Playfair Display, Crimson Text, DM Mono (loaded on first access)
  • Web Speech API — native browser API, no external dependency

Voice Database

Contains 427 voices in 141 locales, manually compiled from public Microsoft Azure Cognitive Services / edge-tts v6+ documentation. It is a static list of metadata — no audio files are included.

Source Code

Full source code is visible in any browser via Ctrl+U. No code is intentionally obfuscated.

Network Requests

On first load, an external request is made to Google Fonts (Playfair Display, Crimson Text, DM Mono). Once cached, the interface works offline.

When any voice is played from the "All Languages" tab, an HTTPS POST request is made to alexio.tf/tts with the sample text and voice name, and the response is an MP3 audio stream. The endpoint can be overridden locally via localStorage.setItem('va_endpoint', 'https://example.com/tts'). The Starter and Storyteller tabs remain 100% local and issue no network requests.
